# 智能体设计模式阅读说明

这组笔记来自对 Jimmy Song《智能体设计模式：智能系统构建实战指南》的学习整理。原书按模式逐章展开，从提示链、路由、并行化、反思、工具使用、规划，到多智能体协作、记忆、MCP、RAG、A2A、安全、评估和探索发现等主题。

我没有按原书逐章复述，而是按"怎么设计一个能落地的 Agent 系统"重新组织为五个篇章：

1. **从概念到落地：智能体全景图**：界定 Agent 和普通 LLM 调用的差别，以及从想法到可上线 Agent 的完整检查清单。
2. **工作流骨架：提示链、路由、并行与规划**：最小可控 Agent 的骨架模式，以及如何组合使用。
3. **接入外部世界：工具、MCP 与知识检索**：Agent 如何通过工具、协议和知识检索接触外部世界。
4. **认知与协作：记忆、反思与多智能体**：Agent 如何跨轮次改进，以及什么时候拆分角色、怎么避免协作失控。
5. **工程化：可靠性、安全、资源与评估**：把 demo 变成可运营系统需要的护栏、异常处理、资源管理和评估体系。

## 我的阅读原则

这组笔记更偏工程视角，而不是概念介绍。每个模式我都按三个问题理解：

- 它解决什么失控点？
- 它会引入什么新复杂度？
- 什么时候应该用，什么时候不该用？

Agent 设计最容易犯的错，是把"模型能多轮调用工具"误认为"系统已经具备智能体能力"。真正的设计重点不是让模型做更多事，而是给它足够清晰的目标、边界、反馈、恢复机制和验证机制。
