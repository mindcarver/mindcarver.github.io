## 核心概念

### Workflow 复盘

Workflow 是这一周的起点，也是整个 Agent 系统的骨架。

回顾 Workflow 的核心要点。

Workflow 的本质是把复杂任务拆成有序步骤，按预定义规则执行。它的三个核心特征是：流程预定义、输入输出明确、状态在节点间传递。

Workflow 的设计方法：拆任务、定输入输出、画流程图、加防护。这四步法适用于任何 Workflow 设计场景。

Workflow 和 Agent 的关系：Workflow 是"计划好的流程"，Agent 是"能自己决定流程的系统"。企业应用中常见"Workflow 包裹 Agent"的混合模式——大框架用 Workflow 控制主流程，需要灵活判断的环节嵌入 Agent。

这一周你应该建立的核心认知是：Workflow 不是 Agent 的简化版，而是和 Agent 互补的架构模式。在任务明确、流程固定的场景下，Workflow 比 Agent 更可控、更经济、更容易调试。不要因为 Agent 更"高级"就不用 Workflow。

如果你在 Workflow 部分有以下疑问，说明理解还不够深入：不清楚什么时候用 Workflow、什么时候用 Agent；说不清节点粒度怎么确定；没有理解状态传递中上下文膨胀的问题。

### Agent 复盘

Agent 是这一周的核心主题。

回顾 Agent 的核心要点。

Agent 的本质是"目标驱动的自主决策系统"。它能感知环境、做出决策、执行动作、根据反馈调整行为。

Agent 和 Chatbot 的区别：Chatbot 是"你问我答"的被动系统，Agent 是"你给目标我来完成"的主动系统。

Agent 的核心组件：Goal（目标）、State（状态）、Memory（记忆）、Tools（工具）、Planner（规划器）、Executor（执行器）、Reflection（反思）。

Agent 的执行循环：Plan → Execute → Reflect → 检查目标是否完成 → 未完成则继续循环。循环的终止条件包括：目标完成、达到最大步数、达到成本上限、遇到不可恢复错误、人工干预。

Agent 的五大失败模式：无限循环、目标漂移、工具误用、幻觉放大、资源耗尽。每种失败模式都有对应的防护措施。

### Tool Calling 复盘

Tool Calling 让 Agent 从"会说话"进化到"能做事"。

回顾 Tool Calling 的核心要点。

Tool Schema 包含四个部分：名称（简洁有意义的命名）、描述（做什么、适用场景、不适用场景）、参数（JSON Schema 定义）、输出（稳定的格式）。

工具选择基于工具描述和当前任务上下文，本质上是模型推理。

参数传递要注意类型、格式、必填项、合理性。错误返回要清晰，告诉模型"什么错了"和"为什么错"。

权限控制三个层次：工具白名单、参数限制、操作审批。

审计日志记录所有调用，用于调试和合规。

### Multi-Agent 复盘

Multi-Agent 让多个专业 Agent 协作完成复杂任务。

回顾 Multi-Agent 的核心要点。

Multi-Agent 的价值：解决单 Agent 的三个痛点（任务太复杂、工具太多、上下文太长）。代价是系统复杂度增加、通信成为新故障点、Token 消耗更大。

主控 Agent（Orchestrator）负责：任务分解、执行调度、结果整合、异常处理。Orchestrator 不应该有自己的专业分析能力。

专家 Agent 设计三要素：职责边界清晰、工具精简、Prompt 专注。

Agent 间通信有间接（通过 Orchestrator）和直接两种，实践中通常是混合模式。

冲突解决策略：优先级规则、人工仲裁、协商机制。

结果整合要做三件事：检查完整性、解决冲突、生成连贯输出。

### Human-in-the-loop 复盘

Human-in-the-loop 是 Agent 可控性的关键。

回顾 Human-in-the-loop 的核心要点。

人工审核的三个原因：模型输出不完全可信、决策涉及主观偏好、责任归属需要明确。

判断审核节点的三条标准：后果严重性、主观偏好、外部影响。

审核表单包含：关键信息摘要、审核选项、审核意见。要简洁，一屏看完。

中断与继续机制：通过、拒绝、要求修改三种结果。

执行记录要完整，用于责任追溯和系统优化。

可控性设计五个原则：目标边界清晰、工具权限控制、步数限制、成本上限、人工介入权。

### Agent 安全复盘

Agent 安全是企业交付的前提。

回顾 Agent 安全的核心要点。

Prompt Injection 分三种类型：直接注入、间接注入、角色注入。防护三道防线：输入过滤、Prompt 加固、输出审查。

工具越权防护三层：工具白名单、参数限制、操作审批。

无限循环防护：最大步数、重复检测、成本上限。

目标漂移防护：定期目标对齐、中间节点审核、输出一致性检查。

幻觉控制：数据溯源、交叉验证、置信度标注。

资源耗尽防护：Token 限制、API 调用限制、状态清理、时间限制。

### Week 4 综合考试

#### 考试题目：安全的 Multi-Agent 行业研究系统设计

**任务：** 设计一个完整的、安全的 Multi-Agent 行业研究系统。

**要求：**

1. **架构设计（40 分）**
   - 定义 Orchestrator 和至少 5 个专家 Agent
   - 明确每个 Agent 的职责边界、工具列表、Prompt 重点
   - 画出执行流程图，标注并行和串行关系
   - 说明状态传递机制和通信方式

2. **工具设计（30 分）**
   - 为至少 5 个工具设计完整的 Tool Schema
   - 定义工具白名单和参数限制
   - 设计操作审批机制

3. **安全防护（30 分）**
   - 写出 Prompt Injection 防护三道防线
   - 设计最大步数、成本上限、时间限制
   - 列出完整的安全防护清单

4. **人工审核设计（20 分）**
   - 识别必须人工审核的节点
   - 设计至少两个审核表单
   - 说明中断与继续机制

5. **交付物（30 分）**
   - 一份完整的系统设计文档（5-10 页）
   - 一份安全防护清单
   - 三套攻击测试样例及防护方案

**评分标准：**
- 架构合理性：Agent 分工是否清晰、职责边界是否明确
- 设计完整性：是否涵盖所有核心组件和防护措施
- 可行性：设计是否能落地实施、是否考虑实际约束
- 安全性：安全措施是否全面、是否能应对主要威胁
- 交付质量：文档是否清晰、表单是否实用、测试样例是否有针对性

**参考时间分配：**
- 架构设计：1 小时
- 工具设计：1 小时
- 安全防护：1 小时
- 人工审核设计：30 分钟
- 交付物整理：30 分钟
