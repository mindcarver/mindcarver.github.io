# AI 辅助策略研究

## 目标

- 入职可以高效快速的理解公司已经有的策略细节，避免各种繁琐无效沟通 ✅
- 可以快速搭建自己的策略想法，0-1 快速展示策略到画布（交互-回显到画布都未去做，验已经验证逻辑可行）✅
- AI 基于现有所有策略知识库，相关数据库，给定目标，进行自主推导，模拟研究员角色，真正的 Agent 角色，进行一个策略迭代 loop：
  - 想法→策略→验证→结果分析→优化策略-验证
  - 先基于已知的一些简单策略进行尝试
- 回测结果解读助手，可以根据回测结果结合目前能够提供的各类数据进行关联性分析
  - 主要是数据维度
- AI 基于海量数据进行统计分析，做到自然语言获取到统计数据，例如：
  - 统计当日某竞争者的利润，和 gasfee 的关系，以及图表化的展示，这样的话扩展性很好，想问什么就能得到什么结果

# PRD 需求

## 一：策略解释器（已经验证，可行性高）

- 可以辅助新人理解，也可以加深老人理解

### 不同研究员产出的策略理解

- 基于现有的研究员的策略节点，然后理解策略内容
  - 节点功能理解
  - 节点参数的理解

```jsx
从前端选择目前平台有的策略完整节点，送给 AI 进行理解
```

---

## 二：AI 策略组合（已经验证，可行性高，主要要考虑前端交互）

- 研究员提出一些想法，AI 可以根据现有的节点模块进行自由组合，比较快速的进行一个初步验证，这个部分比较复杂，得考虑交互，但是实用性比较好

### 形式

比如用户说想验证 BTC 波动率是如何影响其他小币种，比如 USUAL/USDT 等，在一段时间的一个价格比率变化，那么 AI 会根据这个进行一个初步的节点组合，返回给前端，快速进行策略迭代

---

## 三：回测结果解读助手

- 这里需要重点研究研究

### 给 AI 的一些数据

- 回测指标
- 盈亏分布
- 交易订单分析

## 四：AI 头脑风暴

- 这部分不再是 req-resp 的一个形式，而是 AI 作为一个完整的智能体，它可以基于现有的所有节点进行自我学习（比如基于 cex-dex 跨所套利的一些论文，策略方案来源于其他人或者其他源，来进行一个自我的回测→结果→重新执行）

---


## 核心交互流程设计

### 场景 A：AI 策略解释（策略节点 -> 解释）

1. **前端**：发送策略 ID 给 AI 系统。
2. **AI 系统**：通过 API 从核心引擎拉取该策略的 JSON 节点图。
3. **AI 系统**：检索"节点知识库"（向量数据库），获取每个节点的功能文档。
4. **AI 系统**：调用 LLM 生成解读，返回前端。

---

### 场景 B：回测结果解读

1. **核心引擎**：完成回测，将结果摘要（非海量原始数据，而是统计摘要）存入中间层。
2. **AI 系统**：读取摘要 + 抽取典型亏损订单。
3. **AI 系统**：利用 Python Data Agent（如 Pandas Query Tool）进行横向对比（同期其他币种表现）。
4. **AI 系统**：产出结论，推送给研究员。
