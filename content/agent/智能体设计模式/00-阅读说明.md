# 智能体设计模式阅读说明

这组笔记来自对 Jimmy Song《智能体设计模式：智能系统构建实战指南》的学习整理。原书按模式逐章展开，从提示链、路由、并行化、反思、工具使用、规划，到多智能体协作、记忆、MCP、RAG、A2A、安全、评估和探索发现等主题。

我没有按原书逐章复述，而是按“怎么设计一个能落地的 Agent 系统”重新拆成几个篇章：

1. **智能体是什么**：先界定 Agent 和普通 LLM 调用的差别。
2. **基础工作流模式**：提示链、路由、并行化、规划，是最小可控 Agent 的骨架。
3. **工具、MCP 与知识检索**：Agent 如何接入外部世界。
4. **记忆、学习与反思**：Agent 如何跨轮次、跨任务改进。
5. **多智能体协作与通信**：什么时候拆分角色，怎么避免协作失控。
6. **可靠性、安全与评估**：把 demo 变成可运营系统需要哪些护栏。
7. **资源、推理与探索**：成本、延迟、推理深度和开放式探索如何权衡。
8. **落地检查清单**：从想法到可上线 Agent 的设计审查表。

## 我的阅读原则

这组笔记更偏工程视角，而不是概念介绍。每个模式我都按三个问题理解：

- 它解决什么失控点？
- 它会引入什么新复杂度？
- 什么时候应该用，什么时候不该用？

Agent 设计最容易犯的错，是把“模型能多轮调用工具”误认为“系统已经具备智能体能力”。真正的设计重点不是让模型做更多事，而是给它足够清晰的目标、边界、反馈、恢复机制和验证机制。
